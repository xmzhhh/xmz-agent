# 渐进式开发路线图

建议以每周 8～12 小时计算，用 12～16 周完成。时间可以伸缩，但不要跳过验收、测试和复盘。

## 第 0 阶段：项目立项与 Git 起步（1～2 天）

**产物**：需求边界、架构、路线图、学习日记模板、本地 Git 仓库。

**学习**：README、`.gitignore`、工作区/暂存区/提交、规范化提交信息。

**验收**：能解释项目解决什么问题、不解决什么问题；能查看变更和提交历史。

## 第 1 阶段：最小可运行 CLI Agent（1 周）

**功能**：用户输入问题；模型可以调用模拟行情与资产计算工具；输出结构化结论。

**技术**：Python 项目结构、环境变量、Pydantic Schema、异步 API、Tool Calling、最小 ReAct 循环、异常与重试、pytest。

**刻意练习**：不使用 Agent 框架，亲手实现一次“模型 → 工具 → 工具结果 → 模型”的闭环。

**验收**：至少 15 个单元测试；Key 不进入仓库；模型回答无法篡改计算结果。

**Git**：`feat/cli-agent` 分支、原子提交、第一次合并和 `v0.1.0` 标签。

## 第 2 阶段：真实数据适配与资产面板（1～2 周）

**功能**：模拟持仓 CRUD；接入合规可用的数据源；计算收益、权重、集中度；生成日报。

**技术**：Provider/Adapter 模式、HTTP 客户端、缓存、限流、超时、重试、数据新鲜度、FastAPI、简单 UI。

**验收**：断网、限流和字段缺失时能优雅失败；每个数字带数据时间；数据源可用假实现替换。

**Git**：Issue 驱动开发、功能分支、PR 描述、`v0.2.0`。

## 第 3 阶段：记忆与持久化（1～2 周）

**功能**：保存会话、用户风险偏好、关注列表、研究历史和反馈；支持查看、修改和删除记忆。

**技术**：SQLite、SQLAlchemy、Alembic；短期/长期、情景/语义记忆；记忆抽取、冲突更新、TTL 与隐私。

**验收**：重启后状态仍在；用户可解释“系统记住了什么”；敏感字段不进入日志。

**Git**：数据库迁移提交、回滚演练、`v0.3.0`。

## 第 4 阶段：金融知识 RAG 与可验证引用（2 周）

**功能**：导入公开报告和新闻；切分、索引、检索；回答中引用原文、来源和日期。

**技术**：文档解析、元数据、Embedding、稠密/关键词混合检索、过滤、重排、引用对齐、RAG 评测。

**验收**：建立不少于 50 条问题的评测集；报告 Recall@k、MRR、引用正确率和忠实度，而不只展示聊天截图。

**Git**：用 Issue 记录一次检索失败案例与修复，发布 `v0.4.0`。

## 第 5 阶段：上下文工程与可靠工作流（1～2 周）

**功能**：研究任务分解为规划、取数、检索、计算、写作、校验和人工批准节点；支持中断恢复。

**技术**：上下文选择与压缩、token 预算、Prompt 版本、缓存；LangGraph 状态、检查点、幂等、Human-in-the-loop。

**验收**：长对话不无限增长；故障恢复后不重复产生副作用；调仓草稿必须停在批准节点。

**Git**：Architecture Decision Record（ADR）、`v0.5.0`。

## 第 6 阶段：本地模型与模型路由（1 周）

**功能**：通过 Ollama 调用本地模型；根据任务、隐私、延迟和预算选择模型；失败时降级。

**技术**：量化与显存/内存、上下文长度、并发、本地推理 API、模型能力探测、结构化输出兼容性。

**验收**：同一评测集比较云端与本地模型的成功率、质量、首 token 延迟、总延迟和成本。

**Git**：实验报告与 `v0.6.0`。

## 第 7 阶段：MCP 工具生态（1 周）

**功能**：把资产查询或研究资料检索封装成自建 MCP Server；Agent 作为 MCP Client 动态发现与调用工具。

**技术**：Tools/Resources/Prompts、stdio/HTTP transport、能力协商、Schema、超时、权限与安全边界。

**验收**：MCP 服务可被独立客户端调用；恶意参数和越权请求被拒绝；调用可审计。

**Git**：接口契约测试、`v0.7.0`。

## 第 8 阶段：深度研究与多智能体（1～2 周）

**功能**：Planner、Researcher、Risk Analyst、Reporter 分工完成事件研究；保留单 Agent 基线。

**技术**：任务分派、共享状态、并发、结果合并、冲突处理、终止条件。

**验收**：用评测数据证明多 Agent 在某些复杂任务上优于单 Agent，并报告额外延迟与成本；若没有收益，则保留更简单方案。

**Git**：性能对比报告、`v0.8.0`。

## 第 9 阶段：生产化与求职包装（2 周）

**功能**：定时简报、失败重试、监控面板、部署、Demo、完整文档。

**技术**：结构化日志、Trace、指标、成本；Docker Compose；GitHub Actions；安全扫描；单元/集成/契约/E2E 测试。

**特色评测**：选取历史事件及当时可见资料进行时间切片回放，避免“偷看未来”；记录证据质量、结论稳定性和风险覆盖率。

**验收**：新电脑按 README 可启动；CI 全绿；3～5 分钟演示视频；架构图、评测报告、复盘与简历描述齐全；发布 `v1.0.0`。

## 与 Hello-Agents 的穿插学习

- 第 1 阶段前：学习第 4 章经典范式，并自己实现最小循环。
- 第 2～3 阶段：按需学习第 6～8 章，不必先读完再开工。
- 第 4～5 阶段：精读第 8、9、12 章。
- 第 7 阶段：精读第 10 章。
- 第 8 阶段：参考第 13、14、16 章，但不要照搬案例结构。

原则是“项目遇到问题 → 带着问题读对应章节 → 实现 → 评测 → 写复盘”。
