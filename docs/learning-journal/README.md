# 学习日记

每个阶段至少写一篇 Markdown 日记，文件名使用：

```text
01-cli-agent.md
02-data-and-portfolio.md
03-memory-and-persistence.md
...
```

日记不是流水账。它要能帮助未来的你复习面试，也让面试官看到你的判断过程。

## 模板

```markdown
# 第 N 阶段：标题

日期：YYYY-MM-DD  
版本：v0.x.0  
相关 Issue / PR：

## 本阶段目标

## 我实现了什么

## 核心原理（用自己的话）

## 关键设计与取舍

## 遇到的问题

### 问题 1
- 现象：
- 假设：
- 排查证据：
- 根因：
- 修复：
- 如何防止复发：

## 测试与评测结果

## Git / GitHub 新知识

## 本阶段面试题

## 仍然不懂的地方

## 下一阶段计划
```

## 写作要求

- 记录失败尝试，失败往往比最终代码更能体现能力。
- 技术名词必须配自己的解释和一个项目内例子。
- 评测写条件、数据和数字，不写“效果很好”。
- 阶段结束时整理 3～5 个面试问题并口头回答一次。
