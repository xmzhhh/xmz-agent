## 核心原理（用自己的话）

大模型不会直接调用 Python 函数。应用先把工具名称、用途和参数 JSON Schema 发给模型；
模型返回“想调用哪个工具以及参数是什么”；应用再校验参数、找到对应 Python 实现并执行。
因此 Function Calling 本质上是一套由模型提出调用请求、由应用掌握最终执行权的协议。

注册中心像一个受控的工具目录。Agent 只能执行目录中显式注册的工具，而不是根据模型
输出随意执行函数名。这既降低耦合，也形成安全白名单。

## 数据流

1. 具体工具的 Pydantic 输入模型自动生成 JSON Schema。
2. 注册中心把全部 `ToolDefinition` 交给 Agent。
3. Agent 将工具定义随 `ModelRequest` 发给模型。
4. 模型返回 `ToolCall(name, arguments)`。
5. 注册中心按名称查找工具。
6. `BaseTool.execute()` 用输入模型校验参数。
7. 具体工具的 `run()` 执行业务逻辑。
8. `ToolResult` 序列化成 JSON，下一阶段作为 `role=tool` 消息返回模型。

## 关键设计与取舍

- **Schema 由代码生成**：使用 Pydantic 输入模型生成 JSON Schema，避免手写给模型的规则
  与 Python 实际校验规则逐渐不一致。
- **工具入口统一为异步**：当前工具是本地计算，但未来行情、新闻、数据库工具包含 I/O；
  提前统一为 async 接口能让 Agent 循环保持简单。
- **拒绝额外字段**：模型输出属于不可信输入，拼错或臆造参数不能被静默忽略。
- **注册时拒绝重名**：同一个名称只能对应一个实现，确保分派结果确定。
- **数值计算交给代码**：仓位比例用普通公式计算，而不是依赖 LLM 心算，结果更稳定、可测试。
- **模拟数据显式标记**：模拟行情始终包含 `is_mock=True` 和来源，不能伪装成实时数据。
- **本阶段不修改 CLI**：工具层先独立完成，下一小阶段再实现“模型—工具—模型”的 Agent
  循环，便于定位问题和分步提交。

## 遇到的问题

### 问题 1：PowerShell 首次读取中文源码时出现乱码

- **问题现象**：检查 `schemas.py` 时，终端输出的中文注释显示为乱码。
- **产生原因**：PowerShell 的默认文本读取编码与项目 UTF-8 文件编码没有正确匹配；源文件
  本身没有损坏。
- **排查思路**：换用明确指定 UTF-8 的读取方式，再对比同一文件内容。
- **解决方法**：检查时使用 `Get-Content -Encoding utf8`，确认中文内容正常。
- **学到的知识**：终端显示乱码不等于文件内容损坏。应先区分“文件编码”和“终端解码”
  两个环节，不要直接重写源文件。

### 问题 2：pytest 默认临时目录曾出现 Windows 权限错误

- **问题现象**：阶段开始前执行全部测试时，30 个测试通过，但 pytest 访问用户临时目录
  `pytest-of-xmz` 时出现 `WinError 5`。
- **产生原因**：该临时目录的访问权限受当前运行环境限制，不是业务代码失败。
- **排查思路**：观察失败发生在测试收集/临时目录而非断言，并将 TEMP 与 pytest 缓存切换
  到项目内目录进行对照。
- **解决方法**：验证时使用项目内 `.codex_tmp` 作为临时目录和 pytest 缓存目录。
- **学到的知识**：测试基础设施错误要与代码断言失败分开判断；改变临时目录是有效的隔离
  验证手段。

### 问题 3：Ruff 拒绝旧式泛型声明并要求整理格式

- **问题现象**：首次质量检查报告 `UP046`，指出 `BaseTool` 使用了
  `Generic[InputT]`；同时两个新文件需要自动格式化，导入块也不符合 Ruff 顺序。
- **产生原因**：项目最低版本是 Python 3.12，Ruff 因此要求使用 PEP 695 的新泛型语法；
  新增文件在初稿阶段也尚未经过项目统一格式化。
- **排查思路**：根据 Ruff 给出的规则编号定位到泛型类声明，确认 Python 版本和 mypy 均
  支持新语法，再修改声明并重新执行全部质量门禁。
- **解决方法**：把 `class BaseTool(ABC, Generic[InputT])` 改为
  `class BaseTool[InputT: ToolInput](ABC)`，使用 `type AssetCode = ...` 声明类型别名，并让
  Ruff 统一格式和导入顺序。
- **学到的知识**：Ruff 不只检查空格和行宽，也会依据目标 Python 版本提示现代语法；
  自动格式化、lint、类型检查和测试解决的问题不同，四项都需要执行。
