# FinAgent 项目协作约定

## 教学型中文注释

- 本项目后续新增或修改的 Python 代码必须包含清晰、详细的中文注释，帮助初学者理解实现思路。
- 每个模块应使用中文模块文档字符串说明职责、边界以及它在项目中的位置。
- 类、公开函数和关键内部函数应使用中文文档字符串解释用途、参数、返回值、异常和重要约束。
- 对 Agent 循环、模型调用、工具调用、数据校验、异步流程、重试、持久化、RAG、上下文工程和 MCP 等关键逻辑，应在代码附近用中文解释“为什么这样设计”。
- 测试代码应说明测试场景、预期行为以及该测试防止哪类问题。
- 注释必须与代码保持一致；修改行为时同步更新相关注释。
- 避免逐字翻译语法或重复显而易见的代码。注释应帮助理解意图、流程、边界和取舍，不能用大量无信息注释淹没核心逻辑。

## 阶段学习日记

- 每个开发阶段都必须在 `docs/learning-journal/` 中创建或更新对应的 Markdown 学习日记。
- 本阶段开发过程中遇到的所有问题都要记录，不能只记录最终成功结果。
- 每个问题至少应包含：问题现象或错误信息、产生原因、排查思路、解决方法以及从中学到的知识。
- 对尚未解决的问题，应明确记录当前结论、已经尝试的方法和下一步排查方向；解决后及时补充最终结果。
- 阶段结束前应检查代码、测试结果和学习日记是否一致，避免日记遗漏重要的设计调整或故障排查过程。

## 阶段分支与收尾流程

- 每个开发阶段只使用一个功能分支，允许在同一分支中按小模块多次提交和推送。
- 代码、测试、验收脚本、README、架构文档和学习日记全部完成后，才创建该阶段唯一的 Pull Request。
- 创建 PR 后若需要补充 PR 编号或交付元数据，应继续在原功能分支提交，不再创建额外的收尾分支。
- 合并 PR 后只同步 `main`、清理功能分支并发布版本标签；阶段结束前必须确认学习日记已经完成收尾。
